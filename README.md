# クラフトのもり 🌲

**おとぎの国のハンドメイドレシピ帳。** 小さなレシピ帳から、作りたいものを探そう。

初心者でも迷わずハンドメイドを始められる、やさしいレシピ一覧アプリです。
料理レシピアプリのハンドメイド版のようなイメージで、「ジャンル」「難易度」「やりたいこと」から作品を探せます。

モバイルファーストで作られた、下部タブ付きの WEB アプリです。

## デザインの方向性

**「おとぎの国のハンドメイドレシピ帳」** をテーマに、開いた瞬間に世界観が伝わることを目指しています（実用性70% / 世界観30%）。

- 背景は真っ白ではなく、生成り・紙っぽい色＋うっすらドット＋小さな星＋森のやわらかい光
- 見出しは絵本・手書き風フォント（Klee One）、本文は読みやすいゴシック（Zen Kaku Gothic New）、文字色は濃いブラウン
- 配色は「生成り」をベースに、くすみオレンジ／ゴールド／ブラウン＋淡いサブカラー（原色・派手なグラデーション・暗い魔法感は不使用）
- カードは羊皮紙／魔法の本のページ風。しおり（リボン）・ポラロイド写真・木の看板・紙のラベル・付せんで表現
- ジャンルは「森の看板」として、おとぎの国風の名前つき（例：レジン →「きらめきの泉」）
- むずかしさは **はじめて / かんたん（★☆☆）/ ふつう（★★☆）/ じっくり（★★★）** で表示
- 参考画像を大きく、道具はチェックリスト、代替道具は「代わりに使えるもの」とやさしく表示
- お気に入りは「わたしのレシピブック」、作った作品は「作品ノート」に記録
- 下部タブは おうち / さがす / レシピ帳 / 作品ノート / じぶん の5つ

## 主な特長

- 📱 **モバイルファースト** — スマホ表示を最優先。横スクロールしない設計
- 🔠 **文字サイズ設定** — 初回起動時に「小 / 標準 / 大」から選択。あとからマイページで変更可能
- 🧒👵 **子どもから高齢者まで** — 大きめの押しやすいボタン、はっきりしたコントラスト、専門用語は少なめ
- 🔍 **3 つの探し方** — ジャンル / 難易度 / やりたいこと（目的）から検索
- 🖼️ **画像つきの作り方** — 1 ステップ 1 動作で、参考画像と説明文を横並びに表示
- 🛠️ **代替できる道具** — 「ピンセット → 竹串でも可」のように、おうちにあるもので代用する方法と注意点を表示
- ❤️ **お気に入り（わたしのレシピブック）** — ログイン不要。ブラウザに保存（ログイン機能なし）
- 📓 **作品ノート** — 作った作品を「作ったよ」と記録。日付とひとことメモを残せる（ブラウザに保存）

## 使い方の流れ

1. はじめて開くと、「もりの扉」を開くオープニング → 読みやすい文字サイズを選ぶ画面、と進みます（初回のみ。2回目以降は直接ホームへ）
2. ホームで、やりたいこと・難易度・ジャンルから探します
3. 一覧から気になる作品を選びます
4. 完成画像・材料・道具・代替道具を確認します
5. 画像つきの作り方ステップを見ながら作ります
6. 気に入ったレシピは ❤️ で保存します

## 技術スタック

- [React 19](https://react.dev/) + [TypeScript](https://www.typescriptlang.org/)
- [Vite](https://vite.dev/) （開発サーバー / ビルド）
- [Tailwind CSS v4](https://tailwindcss.com/)
- [React Router v7](https://reactrouter.com/) （画面遷移）

バックエンドはありません。すべてブラウザ内で動作し、以下を `localStorage` に保存します。

| キー | 内容 |
| --- | --- |
| `hm_onboarded` | 初回文字サイズ設定が済んだか |
| `hm_font_size` | 選んだ文字サイズ（`small` / `standard` / `large` ＝ 小 / 標準 / 大） |
| `hm_favorites` | お気に入り（わたしのレシピブック）のレシピ ID 一覧 |
| `cn_worknotes` | 作品ノートの記録（作品名・日付・メモ） |

## 開発

```bash
npm install      # 依存関係のインストール
npm run dev      # 開発サーバー起動（http://localhost:5173）
npm run build    # 型チェック + 本番ビルド（dist/）
npm run preview  # 本番ビルドのプレビュー
```

## デプロイ（Vercel）

このアプリは Vite + React の静的 SPA です。Vercel にそのまま公開できます。

- Framework Preset：**Vite**（自動検出）
- Build Command：`npm run build`
- Output Directory：`dist`
- ルーティング：`vercel.json` にて、すべてのパスを `/index.html` に rewrite（React Router のディープリンク対策。`/recipe/xxx` を直接開いても 404 になりません）

設定は [vercel.json](vercel.json) に記載済みのため、リポジトリを Vercel に接続するだけでデプロイできます。
バックエンド・環境変数は不要です（状態はすべてブラウザの `localStorage`）。

## 画像について

本物の写真は使わず、パステルのグラデーション背景＋絵文字による「仮のイメージ画像」で表現しています
（完成画像・各ステップの参考画像・材料 / 道具のイメージ画像）。
実画像に差し替える場合は、`src/components/Thumb.tsx` を画像表示に変更し、`src/data/recipes.ts` に画像 URL を追加してください。

## レシピを追加する

`src/data/recipes.ts` の `RECIPES` 配列にオブジェクトを追加するだけで、一覧・検索・詳細にすぐ反映されます。
新しいジャンルや目的を増やす場合は、同ファイルの `GENRES` / `PURPOSES` と、`src/lib/icons.ts` の絵文字マップも更新してください。
